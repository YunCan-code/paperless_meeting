# 会议室纯内网离线部署说明

本文档适用于单机版会议室部署：一台固定 IP 的 Linux 主机，在纯内网环境中运行 `frontend + backend + PostgreSQL + Redis` 四个服务，不依赖公网拉代码、拉镜像或在线安装依赖。

## 1. 目标与原则

- 会议室主机只负责运行、备份、升级、回滚，不负责源码构建。
- 联网构建机提前完成镜像构建、冒烟验证、镜像导出和离线包打包。
- 会议室主机通过 `docker load` 导入镜像，再通过 `docker compose up -d` 启动。
- 默认目标系统为 `Linux x86_64`，并按 Debian / Ubuntu 系离线安装 Docker 运行时设计。

## 2. 仓库中新增的离线部署资产

### 2.1 根目录

- `compose.offline.yml`
  - 离线专用编排文件，只使用固定 `image:`，不再走 `build:`。
- `compose.offline.env.example`
  - 离线环境变量模板，提供版本号、镜像名、安装路径、数据库口令等参数。

### 2.2 脚本目录

`scripts/intranet/` 下提供以下脚本：

- `prepare-release.sh <version>`
  - 在联网构建机上构建镜像、执行冒烟验证、导出镜像 tar、生成版本化离线包。
- `install-offline.sh`
  - 在会议室主机首次离线安装 Docker 运行时、校验包、导入镜像并启动服务。
- `load-images.sh`
  - 导入 `images/` 下所有镜像 tar。
- `start.sh`
  - 启动全部服务。
- `stop.sh`
  - 停止全部服务。
- `status.sh`
  - 查看当前版本、镜像和容器状态。
- `backup.sh [label]`
  - 备份 PostgreSQL 数据库和上传文件。
- `restore.sh <backup_dir> [--yes]`
  - 用指定备份恢复数据库和上传文件。
- `upgrade.sh [version]`
  - 先备份，再导入镜像并重建前后端。
- `verify-checksums.sh`
  - 校验离线包中文件的 SHA256。

## 3. 联网构建机发布流程

### 3.1 前置条件

- 已安装 Docker 和 Docker Compose 插件
- 能访问 Docker Hub、PyPI、npm 镜像源
- 当前代码处于可构建状态

### 3.2 执行命令

```bash
cd /path/to/paperless_meeting
bash ./scripts/intranet/prepare-release.sh 2026.04.08
```

默认行为：

- 构建镜像：
  - `paperless-meeting/backend:2026.04.08`
  - `paperless-meeting/frontend:2026.04.08`
- 拉取基础镜像：
  - `postgres:15-alpine`
  - `redis:7-alpine`
- 启动一套临时环境做本地冒烟验证
- 导出 4 个镜像 tar
- 生成离线目录：

```text
release/2026.04.08/
├─ images/
├─ compose/
├─ scripts/
├─ docs/
├─ runtime/
├─ checksums/
└─ VERSION
```

- 生成压缩包：
  - `release/paperless-meeting-offline-2026.04.08.tar.gz`

### 3.3 Docker 运行时离线包

`prepare-release.sh` 不会自动下载 Docker 安装包。  
请在联网构建机打包前，先把目标主机所需的 `.deb` 文件放进仓库根目录中的：

```text
runtime/docker/debs/
```

建议包含：

- `containerd.io_*.deb`
- `docker-ce-cli_*.deb`
- `docker-ce_*.deb`
- `docker-buildx-plugin_*.deb`
- `docker-compose-plugin_*.deb`

打包脚本会自动把这些文件复制到发布包中的：

```text
release/<version>/runtime/docker/debs/
```

## 4. 会议室主机首次安装

### 4.1 拷贝离线包

把 `release/<version>/` 整个目录，或对应压缩包内容，复制到会议室主机，例如：

```bash
/opt/paperless_meeting
```

目录结构应类似：

```text
/opt/paperless_meeting/
├─ images/
├─ compose/
├─ scripts/
├─ docs/
├─ runtime/
├─ checksums/
└─ VERSION
```

### 4.2 修改环境文件

编辑：

```bash
/opt/paperless_meeting/compose/.env.offline
```

至少确认这些值：

- `APP_ROOT=/opt/paperless_meeting`
- `POSTGRES_PASSWORD=<强密码>`
- `FRONTEND_PORT=5000`
- `CORS_ORIGINS=http://<会议室主机IP>:5000,http://127.0.0.1:5000,http://localhost:5000`

### 4.3 执行首次安装

```bash
cd /opt/paperless_meeting/scripts
sudo bash ./install-offline.sh
```

该脚本会依次执行：

- 校验 SHA256
- 安装 Docker 运行时（若主机尚未安装）
- 导入镜像 tar
- 创建 `data/`、`backups/` 目录
- 启动四个服务

安装完成后，访问：

```text
http://<会议室主机IP>:5000
```

## 5. 日常运维

在会议室主机上执行：

```bash
cd /opt/paperless_meeting/scripts
```

常用命令：

```bash
bash ./status.sh
bash ./backup.sh manual
bash ./stop.sh
bash ./start.sh
```

备份文件默认位于：

```text
/opt/paperless_meeting/backups/
```

## 6. 离线升级

升级时不再执行 `git pull` 或 `docker compose build`。

步骤如下：

1. 在联网构建机生成新版本离线包。
2. 把新版本内容覆盖到会议室主机的 `/opt/paperless_meeting`。
3. 编辑新版本的 `compose/.env.offline`，确认 `POSTGRES_PASSWORD`、`APP_ROOT`、`CORS_ORIGINS` 未丢失。
4. 执行：

```bash
cd /opt/paperless_meeting/scripts
bash ./upgrade.sh 2026.04.09
```

升级脚本会：

- 先创建一份升级前备份
- 导入新镜像
- 更新 `APP_VERSION`、`BACKEND_IMAGE`、`FRONTEND_IMAGE`
- 保留 PostgreSQL 和 Redis 数据卷
- 仅重建 `backend`、`frontend`

## 7. 回滚与恢复

### 7.1 基于备份恢复数据

```bash
cd /opt/paperless_meeting/scripts
bash ./restore.sh /opt/paperless_meeting/backups/20260408-203000-pre-upgrade-2026.04.09
```

### 7.2 基于旧版本回滚应用

- 保留旧版本离线包
- 重新覆盖旧版本的 `images/`、`compose/`、`scripts/`
- 执行：

```bash
bash ./upgrade.sh <旧版本号>
```

## 8. 验收建议

建议在断网虚机中完成至少一轮全流程验证：

- 空机离线安装 Docker 并启动项目
- 登录后台，创建会议
- 上传 PDF
- 发起投票并查看大屏结果
- 发起抽签并验证状态同步
- 重启容器、重启宿主机后验证数据仍在
- 升级一个新版本，再验证可回滚

## 9. 当前假设

- 目标主机为 Linux x86_64
- 目标发行版为 Debian / Ubuntu 系
- 生产架构继续保留 PostgreSQL、Redis、backend、frontend 四服务
- 会议室现场不需要在线开发或在线构建能力
